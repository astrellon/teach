Statements

Lets start right at the begining, what is a statement? In the language we're going to look at C# it's everything between two semi-colons (;). These statements make up the 'sentances' of the code, you can combine multiple operations into a single statement, but you don't want them getting too long as it can be awkward to handle. A single statement is not a single line of code either. In this code there are two statements; 

      string name = "Beth";
      int age =
                26;

While the second line is a bit awkward it is still a single statement even though its goes over two lines. Multiline statements are something to be aware of as when a single statement gets too long it can make sense to break it up over multiple lines, or into multiple statements. For now though I'm just going to stick with simple statements that won't need multiple lines.

Declaration Statement

The statement shown above is called a declaration statement, as it declares that a variable exists, with a specific type and an optional starting value. Declaration statments have the general structure one of the two: 

      VariableType1 variableName1;
      VariableType2 variableName2 = startingValue;

Comments

Whilst not quite statements as such, they allow code to contain notes that can be used to describe what the code is doing in human language. These are useful just for you as the developer to keep track of what you've written and important for other developers who are looking at your code. 

    // This is a single line, anything after the // is considered a comment until a new line is reached.

Variables

At the high level variables can be thought of as boxes with labels that contain data. Each box has a type associated with it and that box can only store values that are of that type in it.

Variable Types

There are two main groups of types, primative and complex (sometimes called compound types). Lets just look at primative types first:

Now what do they mean!? Well internally in the computer each of those are just ones and zeroes, generally made up of either 8, 16, 32 or 64 ones and zeroes (ones and zeroes are generallly called bits, eg: 32-bit, 64-bit, etc). So it's up to us as the programmer to say what we want to do with those bits or rather, what those bits mean.

For the sake of simplicity I will stick to using int, float and bool for now as the other values are pretty much just variations on those types anymway.
  • int

    int is short for integer which is the math term for a whole number, so no decimal points with this type. They are fast, small, accurate (if you're using whole numbers) and generally one of the most common types. In C# they are 32-bit which means they can store a value range between -2147483648 and 2147483647 or [-2^31, 2^31 - 1]. Why minus one? That's honestly not super important right now, but just know that there are rules associated with how the number is stored and limitations on the size of the numbers.

    For example: 

              int age = 26;
          int year = 2015;
          int pi = 3.14159; // <-- Invalid! Because pi is not an integer value

  • float

    float is short for floating point number, which just means that it's a number that can have a deciaml value. It's a number type that has a certain accuracy, so at large numbers, adding 1 to that large number will not do anything. Or in the reverse case, having a small number and then adding a really big number and then subtracting that big number will result in the small value you had being lost. Again the specifics are not important but know that there are limitations.

    For example: 
            float age = 26; // <-- This is valid because 26 is the decimal number 26.0
        float pi = 3.14159;
        float bigNumber = 1000000000005; // <-- This will be rounded to about 100
  • bool bool is short for boolean which I'm pretty sure is the name of the guy who invented Boolean math, which is the math based around true and false only values. So a bool can only store either true or false, there isn't really any other surprises with bools than that.

Program Flow

So on their own variables don't do anything, you need something to execute instructions. Code is often compared to recipe instructions which is pretty true, however instead of a person interpreting a list of relatively vague instructions you have a computer executing very precise instructions.

Each code statement does one or more things. For example: 
      string name = "Beth";
      int age = 43;
      string email = "example@what.com";

In this case, we have 3 declaration statements. Before any of this code has run the computer has no knowledge the variables name, age or email. As each declaration statement is executed it creates space in memory for each variable and puts the value on the right hand side (after the = sign) into the space owned by those variables.

So each statement is executed one after the other sequentially (not actually 100% true but for now you can assume that is the case). However to do anything complicated you need to be able to re-use and rerun code, or conditionally run code, etc. For doing these things there are control statements:

if statement

The if statement is used to run code if a certain condition is met, eg: 
      int pingTime = 1000;
      string status;
      if (pingTime > 500)
      {
          status = "laggy";
      }
      else
      {
          status = "not too bad";
      } 
      Console.WriteLine(status);

Assuming that the value of pingTime was set by something dynamic instead of something statically set in the code, you use this code to see the current status value about the ping time.

You may have also noticed that there is some more syntax going on, the use of curly brackets {}s and the greater than operator >. The brackets tell the computer which sets of lines of code go together, like in this case you can put as many lines of code as you want within the first and second sets of curly brackets for executing code when the condition is false or true. There are also numerous operators for testing numbers, most are fairly self explanatory with a few exceptions that you probably won't need in most cases.

for loop

Now we're getting into some more complicated statements. The for loop is one of three looping statements but each are pretty much just for preference or easy of use in a given situation, all loops can be used interchangeably really. All loops follow the same basic structure, set some initial values, a condition to be met (like an if statement) and something to perform at the end of each loop iteration. 
      int total = 0;
      for (int i = 0; i < 10; i = i + 1)
      {
          total = total + i;
          Console.WriteLine(total);
      }

So what's going on here!? The for has the three parts to it, firstly is the int i = 0;, this creates a new variable just for this loop (not 100% true, this changes based on language but convention dictates that we assume that the i variable only exists in the for loop). We also have our condition i < 10; which will allow the loop to run while i is less than 10. And finally we have our step i = i + 1, this is just another piece of code that is run each time the loop is executed.

I've also used a function console.info functions are just more code but you can give them inputs, in this case I want to display the current total value to the screen somewhere. Assuming you ran this code in a browser you would see this output in the console. 
      0
      1
      3
      6
      10
      15
      21
      28
      36
      45

You can see that the code was executed ten times and gave us this list of numbers in the console.

I know this can be a lot of new things to take in so don't worry if you're still feeling pretty lost. But hopefully you have a rough idea on what variables are and that program flow is something that can be changed.

Functions

Functions are another key concepts in programming and the concept is entirely taken from the idea of math functions. Thinking back to math, sine, cosine and tan were all functions and using them in code would look like this: 

      double answer = sine(input);

Easy! So lets say we want to make our own function since we'll want to do that frequently. Lets make a really simple function that takes a number, increments the number and returns the new number. 

      int addOne(int input)
      {
          return input + 1;
      }
      int x = 5;
      int y = addOne(x);

So the value of y is now 6. Now there are several important things going on here:

  1. Firstly the value of x is copied into the value of input within the addOne function, since it's another variable that now has the same value, any changes made to the input value has no affect on the original x variable.
  2. Secondly the variables created within the addOne function only exist within the function, so asking for the value of input outside the function will return undefined.

Lets try a slightly more complicated function that takes two arguments (inputs). 

      int z = 8;
      int combine(int x, int y)
      {
          return x * y * z; 
      }

      int x = 5;
      int y = combine(x, 9);

Now the value of y will be 360. You might notice that the names of variables have been used twice, however this doesn't cause problems because the variables inside the function 'shadow' the values outside the function. However variables outside the function can be referred to if a name hasn't been 'shadow'd as in the case of the z variable.

Value Types

Now that we hopefully have some basic concept of variables, assigning the value of variables and some basic program flow, here are some more information about variables. As you saw above there are different types of values, for example "Beth" is a string, short for a string of characters, and 26 is a number.

Now there are generally a few basic primative types that are going to be common to nearly all languages as they relate to how CPUs work generally.

  • Number (usually split into integer and decimal types, but not always)
  • Boolean (true or false values only)
There are also more complex compound types that are essentially just a combination of those twos wrapped into one type, for example a Person type might look like this: 
      var person = {
          name: "Beth",
          age: 26,
          email: "email.com"
      };
      console.info("Person: ", person.name, person.age, person.email);
This would print to the console: 
      Person: Beth 26 email.com

Now here's where things can get tricky to explain as all my example have been in Javascript for simplicity and the ability to use a browser for doing any code tests. However Javascript does like to do things a bit differently from other languages so while the concepts are still the same, the way things are done in the code might appear quite different so bare that in mind.
