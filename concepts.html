Knowing these concepts are the foundation of programming. Learning the syntax of a particular language is the easy part, knowing how to use these concepts is the hard part.

So lets start with some of the basic concepts. Variables and programming flow are kind of both needed at the same time for things to make any sense.

Variables

At the high level variables can be thought of as boxes with labels that contain data. Lets use the common concept of a person and make some variables that can be attributed to a person.

  • Name
  • Age
  • Phone Number
  • Email

Values can be assigned to these variables and then retrieved later, eg: 

name = "Beth";
age = 26;

This is an example of setting some variables with some values, don't worry about the exact syntax just yet. Only note that there are different types of values as "Beth" is a word and 26 is a number.

Okay this is pretty brief on variables since we need to move onto program flow to really make use of variables and we'll come back to the details of variables.

Program Flow

So on their own variables don't do anything, you need something to execute instructions. Code is often compared to recipe instructions which is pretty true, however instead of a person interpreting a list of relatively vague instructions you have a computer executing very precise instructions.

Each line of code does one or more things, these are typically (depending on the language) anything between two semi-colons (;) For example: 
var name = "Beth";
var age = 43;
var email = "example@what.com";
Is the same as 
var name = "Beth"; var age = 43; var email = "example@what.com";

For the purposes of readability and sanity people will write their code so that each statement is on a new line, however if you see multiple statements on the same line it's still valid because it's the semi-colon's that actually separate the lines. You could say that semi-colon's are like periods (.) and that the whitespace around those is for organising things logically.

So each line is executed one after the other sequentially (not actually 100% true but for now you can assume that is the case). However to do anything complicated you need to be able to re-use and rerun code, or conditionally run code, etc. For doing these things there are control statements:

if statement

The if statement is used to run code if a certain condition is met, eg: 
var pingTime = 1000;
var status;
if (pingTime > 500)
{
    status = "laggy";
}
else
{
    status = "not too bad";
}

Assuming that the value of pingTime was set by something dynamic instead of something statically set in the code, you use this code to see the current status value about the ping time.

You may have also noticed that there is some more syntax going on, the use of curly brackets {}s and the greater than operator >. The brackets tell the computer which sets of lines of code go together, like in this case you can put as many lines of code as you want within the first and second sets of curly brackets for executing code when the condition is false or true. There are also numerous operators for testing numbers, most are fairly self explanatory with a few exceptions that you probably won't need in most cases.

for loop

Now we're getting into some more complicated statements. The for loop is one of three looping statements but each are pretty much just for preference or easy of use in a given situation, all loops can be used interchangeably really. All loops follow the same basic structure, set some initial values, a condition to be met (like an if statement) and something to perform at the end of each loop iteration. 
var total = 0;
for (var i = 0; i < 10; i = i + 1)
{
  total = total + i;
  console.info(total);
}

So what's going on here!? The for has the three parts to it, firstly is the var i = 0;, this creates a new variable just for this loop (not 100% true, this changes based on language but convention dictates that we assume that the i variable only exists in the for loop). We also have our condition i < 10; which will allow the loop to run while i is less than 10. And finally we have our step i = i + 1, this is just another piece of code that is run each time the loop is executed.

I've also used a function console.info functions are just more code but you can give them inputs, in this case I want to display the current total value to the screen somewhere. Assuming you ran this code in a browser you would see this output in the console. 
0
1
3
6
10
15
21
28
36
45

You can see that the code was executed ten times and gave us this list of numbers in the console.

I know this can be a lot of new things to take in so don't worry if you're still feeling pretty lost. But hopefully you have a rough idea on what variables are and that program flow is something that can be changed.

Functions

Functions are another key concepts in programming and the concept is entirely taken from the idea of math functions. So assuming that you remember that you could get the sine or cosine value of a number and get a different number back, that was a function.
